## Background (zero-FV-expertise refresher)

**Aeneas** (`https://github.com/AeneasVerif/aeneas`) translates Rust to Lean 4 via an intermediate IR called **LLBC** (Low-Level Borrow Calculus) produced by **Charon** (`https://github.com/AeneasVerif/charon`). Charon reads Rust's MIR; Aeneas consumes the LLBC and emits Lean 4. Ownership and mutable borrows are threaded through a state monad in the output â€” the generated code is faithful but verbose. You prove theorems *about* the generated module, not *on* it directly. That is exactly what future Lean Squad tasks (Task 9 onwards) will do.
