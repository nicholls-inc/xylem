- **Every generated artefact carries a 🔬 disclosure.** This is a visible one-line banner at the top of every placeholder markdown file that makes the auto-generation lineage obvious to any human reader. Exact line to use:

## Task

Execute these steps in order. Each step is idempotent: re-running this phase on a half-bootstrapped repo must converge to the same final state.

### Step 1 — Install elan if missing

Check for `elan` with `command -v elan` (or `which elan`). If absent, install it non-interactively without `sudo`, using a hard 300-second timeout:

```bash
if ! command -v elan >/dev/null 2>&1; then
  timeout 300s bash -c 'curl -sSf https://raw.githubusercontent.com/leanprover/elan/master/elan-init.sh | sh -s -- -y --default-toolchain none'
fi
```

Notes:
- `--default-toolchain none` means we do NOT install a Lean toolchain right now. The toolchain pin in `formal-verification/lean-toolchain` (created below) will cause elan to fetch the correct version on first `lake build`.
- `--no-modify-path` is NOT required here; the installer will append to `~/.profile` but will not require an interactive shell restart because we explicitly source `"$HOME/.elan/env"` for later steps.
- No `sudo`. Elan installs entirely under `$HOME/.elan/`.
- After install, source the env for this shell session: `. "$HOME/.elan/env" 2>/dev/null || true`.

If the install times out or fails, do not abort the whole phase — continue to create the scaffolding files. The verify-phase gate will report the missing tool clearly, which is the right signal.
